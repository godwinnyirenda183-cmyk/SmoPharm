# Pharmacy POS

Offline-first pharmacy inventory and point-of-sale application for a small pharmacy in Zambia.

**Platforms:** Windows desktop, Android tablet  
**Stack:** Flutter/Dart · Drift (SQLite) · Riverpod · Supabase

---

## Getting Started

### Prerequisites

- Flutter SDK ≥ 3.3.0 ([install guide](https://docs.flutter.dev/get-started/install))
- For Windows builds: Visual Studio 2022 with "Desktop development with C++" workload
- For Android builds: Android Studio + NDK

### Setup

```bash
cd pharmacy_pos

# Install dependencies
flutter pub get

# Generate Drift database code (required before first build/test)
flutter pub run build_runner build --delete-conflicting-outputs

# Run on Windows
flutter run -d windows

# Run on Android (device/emulator must be connected)
flutter run -d android

# Run tests
flutter test
```

### Code Generation

Drift uses code generation for type-safe queries. After modifying any table in
`lib/database/tables.dart`, regenerate with:

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

## Project Structure

```
lib/
  main.dart                  # App entry point, ProviderScope root
  database/
    tables.dart              # Drift table definitions (all 10 tables)
    database.dart            # AppDatabase class + migration
    database.g.dart          # Generated by build_runner (do not edit)
  providers/
    database_provider.dart   # Riverpod provider for AppDatabase
test/
  database/
    database_test.dart       # Schema smoke tests
```

## Database Schema

All monetary values are stored as **integer cents** (e.g. ZMW 10.00 = 1000).

| Table | Description |
|---|---|
| `users` | Authenticated operators (admin / cashier) |
| `products` | Medicine catalogue |
| `batches` | Stock lots with expiry dates |
| `stock_ins` | Stock receipt events |
| `stock_in_lines` | Individual batch lines within a stock-in |
| `stock_adjustments` | Manual inventory corrections with reason codes |
| `sales` | POS transactions |
| `sale_items` | Line items within a sale |
| `offline_queue` | Transactions queued for remote sync |
| `settings` | Key-value configuration store |
